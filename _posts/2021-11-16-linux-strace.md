## 1) 使用场景

strace 常用来跟踪进程执行时的系统调用和所接收的信号。

在Linux世界，进程不能直接访问硬件设备，当进程需要访问硬件设备时，必须由用户态模式切换至内核态模式，通过系统调用访问硬件设备。

strace可以跟踪到一个进程产生的系统调用，包括参数，返回值，执行消耗的时间。

## 2) 运行模式

strace有两种运行模式：

### 方式1：启动要跟踪的进程

```bash
strace ls -l
```

### 方式2：跟踪已运行的进程

```bash
strace -p pid
```

## 3) 常用参数

- `-c` 统计每条系统调用的次数、时间、错误
- `-f` 跟踪fork的子进程
- `-t` 在输出中的每行加时间戳
- `-T` 显示每次调用花费的时间
- `-o filename` 输出到文件
- `-e expr` 指定表达式过滤
